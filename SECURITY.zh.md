# 安全

Microsoft 非常重视其软件产品与服务的安全性，这也包括通过其 GitHub 组织管理的所有源代码仓库（例如 Microsoft、Azure、DotNet、AspNet、Xamarin）。

如果您认为在任何 Microsoft 拥有的代码库中发现了安全漏洞，请按照下述方式报告：

## 报告安全问题

**请勿通过公开的 GitHub issue 报告安全漏洞。**

请通过 Microsoft 安全响应中心（MSRC）在 https://aka.ms/security.md/msrc/create-report 提交报告。

如果您希望匿名提交或不登录，可发送邮件至 secure@microsoft.com。若可能，请使用我们的 PGP 密钥加密消息（PGP 密钥可从 MSRC 页面下载）。

您应在 24 小时内收到回复；如果未收到，请通过邮件跟进以确保我们已收到您的报告。更多信息见 microsoft.com/msrc。

请尽可能提供以下信息以便我们更快地处理：
- 问题类型（例如缓冲区溢出、SQL 注入、跨站脚本等）
- 涉及的源文件完整路径
- 受影响的代码位置（tag/branch/commit 或 直接 URL）
- 重现该问题所需的特殊配置
- 重现步骤
- 证明概念或攻击代码（如果可能）
- 该问题可能的影响及攻击者如何利用它

完整的报告有助于漏洞奖赏计划（若适用）。更多详情请见 Microsoft Bug Bounty 页面（https://aka.ms/security.md/msrc/bounty）。

## 首选语言

我们偏好英语沟通。

## 策略

Microsoft 采用协同披露（Coordinated Vulnerability Disclosure）原则处理安全问题。
